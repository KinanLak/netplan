# Migration: react-hotkeys-hook → @tanstack/react-hotkeys

## Summary

Replace all keyboard shortcut handling (react-hotkeys-hook + native `addEventListener`) with `@tanstack/react-hotkeys`. Simplify shortcut bindings, fix browser default overrides, implement missing pan shortcuts.

---

## Key Architecture Decisions

### 1. Scope replacement: `enabled` conditions instead of named scopes

TanStack has no named scope system. Replace with `enabled` conditions derived from `selectedDeviceId` (drawer open = device selected):

- **Global**: always enabled
- **Canvas**: `enabled: !drawerOpen` (derived from `selectedDeviceId == null`)
- **Drawer**: `enabled: drawerOpen` (derived from `selectedDeviceId != null`)

`useDrawerScope`, `useScopes`, `useHotkeysContext` are all deleted. The `useScopeEnabled(scope, enabled)` helper centralizes the logic.

### 2. AZERTY/QWERTY handling is built-in for digits

TanStack uses `event.key` with automatic **fallback to `event.code`** for letters (A-Z) and digits (0-9). Pressing the physical `1` key on AZERTY (which produces `&`) still matches hotkey `"1"`. No dual registration needed.

**Exception**: `Shift+digit` is explicitly unsupported by TanStack across layouts.

### 3. Smart `ignoreInputs` default

TanStack's default behavior:

- Single keys (E, W, H, 1, etc.): **ignored in inputs** — correct for tool shortcuts
- Modifier combos (Mod+Z, Mod+B): **fire in inputs** — need `ignoreInputs: true` for undo/redo/sidebar
- Escape: **fires in inputs** — correct for closing dialogs

### 4. `useOptionHeld` stays native

This hook tracks whether a modifier key is held down (with delay + blur handling). It's not a shortcut — it's UI state tracking. TanStack has no `useKeyHeld` primitive. Keep as native `addEventListener`.

## File Changes

- Remove `react-hotkeys-hook`
- Add `@tanstack/react-hotkeys`

### 2. `src/lib/shortcuts.ts`

- Change `hotkey: string | string[]` → `keys: [string, ...string[]]` (TanStack format)
- Update all key format: `mod+z` → `Mod+Z`, `escape` → `Escape`, `up` → `ArrowUp`, etc.
- Remove `floor-1` through `floor-9` actions
- Add `sidebar-toggle` action (move from native handler)
- Simplify tool bindings to letters only for wall tools, digits only for device tools
- Update `formatHotkey()` to parse TanStack capitalized format
- Update `getShortcutDisplay()` to read `config.keys` instead of `config.hotkey`
- Remove `getHotkey()` and `getScope()` (replaced by direct config access)

### 3. `src/lib/shortcut-groups.ts`

- Remove floor-navigation group (only floor-up/down remain, move to General or Zoom/Navigation)
- Update any references to removed actions
- Add `sidebar-toggle` if appropriate

### 4. `src/lib/constants.ts`

- Remove `SHORTCUT_FORM_TAGS` (TanStack handles input detection automatically)
- Remove `SIDEBAR_KEYBOARD_SHORTCUT` (moved to shortcuts config)
- Keep `OVERLAY_VISIBILITY_DELAY_MS`, `OVERLAY_MODIFIER_KEY_BY_PLATFORM` (used by `useOptionHeld`)
- Keep `FLOW_CANVAS_TOGGLE_DEBUG_HOTKEY` → update format to `"Shift+D"`
- Add `PAN_AMOUNT` constant (e.g., 50px) for arrow key panning

### 5. `src/hooks/use-shortcuts.tsx`

Complete rewrite:

```tsx
// Keep: useOptionHeld (native, unchanged)
// New: useShortcut(action, handler, options) — wraps useHotkey with config + scope
// New: useScopeEnabled(scope, enabled) — replaces scope system
// Delete: useHotkeyDirect, useDrawerScope, useScopes, useHotkeysContext re-export
```

**`useShortcut` design**: Calls `useHotkey` once per key binding (fixed 2 slots to satisfy Rules of Hooks). Uses `useScopeEnabled()` to compute `enabled` from scope + drawer state.

**`useScopeEnabled` design**: Reads `selectedDeviceId` from store. Global → always true. Canvas → `!selectedDeviceId`. Drawer → `!!selectedDeviceId`.

### 6. `src/components/hotkeys-provider.tsx`

- Replace `react-hotkeys-hook` `HotkeysProvider` with `@tanstack/react-hotkeys` `HotkeysProvider`
- Set default options: `{ preventDefault: true }`
